(1) “TBA Full Inspection - Issue Reporting App”

========================
5) CLEANER / INSPECTOR APP: HOME & FLOWS
========================
Mobile-first home screen shows:
- “Quick Report”
- “Full Inspection” (for cleaners)
- “Onboarding Inspection” (for inspectors)

A) QUICK REPORT FLOW
Steps:
1. Choose unit (searchable dropdown)
2. Confirm/select Responsible PM (auto-filled if pre-assigned; else required dropdown)
3. Capture photo or video (mandatory)
4. Add short description (required, min 5 chars)
5. Submit

Rules:
- Media required (at least 1)
- Media auto timestamped (timestampCapturedAt = client time; also store server received time)
- On submit:
  - Create QuickReport + Media
  - Create Notification for responsible PM
  - Send email to responsible PM with unit info, timestamp, description, and media links

B) FULL INSPECTION FLOW (CLEANER)
Steps:
1. Choose unit (or pick from “Assigned to me” list)
2. Confirm/select Responsible PM (same rule)
3. Room-by-room checklist UI:
   - Show ONE room at a time
   - Progress indicator (rooms completed / total)
   - “Next Room” disabled until all required items in current room completed
4. Each checklist item:
   - Pass/Fail/N/A (or Yes/No depending on item)
   - REQUIRED media upload for every item (at least 1 photo OR video)
   - Failed items optionally show notes + severity dropdown
5. Cannot submit inspection unless:
   - All rooms completed
   - Every item has result selected
   - Every item has at least 1 media attached
6. Submit creates:
   - ChecklistResponses + Media records
   - Update InspectionTask status to SUBMITTED
   - Create Notification for responsible PM
   - Send full inspection email summary with:
     - unit details
     - timestamp range
     - checklist results breakdown
     - links to media per item

Implement “resume in-progress”: if user closes, they can continue later.

7) ONBOARDING INSPECTION MODULE (INSPECTOR)
========================
Implement a parallel flow “Onboarding Inspection” similar to Full Inspection but with different result options:
For onboarding checklist items, each item can be:
- MISSING
- NEED_REPLACEMENT (requires reason note)
- GOOD_CONDITION
Rules:
- Photo required for NEED_REPLACEMENT and GOOD_CONDITION
- Photo NOT required if MISSING (but allow optional)
- Enforce required inputs (result selection; reason if need replacement)

The onboarding checklist structure is “pending” in spec, so implement:
- A simple template with the same room-based structure
- A section called “Issues & Damages” where inspector can add multiple issue entries:
  - title, description, severity, media required (at least 1)
- The system must generate an “owner-ready report” immediately upon submission:
  - Executive summary (counts of missing, need replacement, good)
  - Room-by-room condition overview
  - Issues & damages list with photos
Output formats:
- Web dashboard report view
- PDF export (server-side PDF generation)

PDF generation:
- Use a Node PDF library (e.g., pdfkit) and embed images as thumbnails with captions.
- Store generated PDF URL/path in DB (InspectionTask.reportPdfUrl)

On submit:
- Notify responsible PM + email them with the report link + PDF link.

Success metrics to support:
- inspection-to-report under 5 minutes (optimize UX, no unnecessary steps)
- zero follow-up due to validation completeness
